# Glossary — Model 1 v1.0

This glossary defines only the public v1.0 vocabulary. Each term is deliberately narrow.

| Term | Definition |
| --- | --- |
| **Model 1 v1.0** | The public name of a minimal stop-loss guardrail protocol. It is not an AI model or execution engine. |
| **Protocol** | A small ordered set of checks used before a consequential next step. |
| **Guardrail** | A boundary-preserving check that can signal risk but cannot make a person's decision. |
| **Safety brake** | A mechanism for slowing, pausing, freezing, or exiting before further cost becomes difficult to reverse. |
| **Pre-execution** | Before the next consequential action is carried out. It may occur within an already ongoing situation; it does not mean speculative prediction. |
| **Consequential next step** | A proposed action that may increase cost, exposure, responsibility, lock-in, or irreversibility. |
| **Stop-loss** | The act of limiting further cost once continuing no longer remains acceptably reversible or boundary-consistent. |
| **Boundary** | A stated constraint that the user is not willing to surrender merely to keep a process moving. |
| **Accumulating cost** | Increasing consumption of time, energy, exposure, responsibility, or integrity. |
| **Reversibility** | The degree to which a person can stop, withdraw, correct, or return to baseline without unacceptable loss. |
| **Signal** | A bounded risk indication; not a prediction, approval, instruction, or final decision. |
| **Green** | No stop-loss trigger is identified from the stated first-person input. No guarantee is provided. |
| **Yellow** | Risk is increasing; slow down, check evidence and boundaries, and preserve options. |
| **Red** | A hard boundary or irreversible-cost risk is present; freeze or exit is a reasonable option, subject to human decision. |
| **Freeze** | A temporary stop on further consequential movement while the user checks boundaries, evidence, and responsibility. |
| **Exit** | A decision by the user to stop participating in the relevant path or process. |
| **Human Final Gate** | The non-transferable requirement that a person makes and owns the final decision. |
| **First-person-only** | The restriction that the protocol evaluates the user's own exposure and next step, not another person's psychology or vulnerabilities. |
| **Anti-weaponization** | The prohibition on converting stop-loss judgment into influence, coercion, profiling, or control tactics. |
| **Editorial decoder** | A limited AI role that may format, translate, clarify, or compress while preserving meaning and boundaries. |
| **Public / private boundary** | Public v1.0 exposes minimal structure and safety limits; private formation material, identifiable cases, thresholds, and execution layers remain excluded. |
